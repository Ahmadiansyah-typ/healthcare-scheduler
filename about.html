About the Project

Detailed information about algorithms, methodology, and research objectives

Project Overview

This research project focuses on optimizing patient-doctor scheduling in cloud healthcare environments. The primary goal is to efficiently assign patients to available doctors while minimizing workload imbalance across all healthcare providers.

The problem is formulated as an optimization task where each patient must be assigned to exactly one doctor, and the objective is to minimize the standard deviation of total consultation times across all doctors. Lower standard deviation indicates better workload balance and more equitable distribution of patients.

Problem Definition

Mathematical Formulation

Input:

  • nP: Number of patients (100, 200, 300, or 400)
  • nD: Number of doctors (10, 20, 30, or 40)
  • patient_times[i]: Consultation time for patient i (random: 5-20 minutes)

Decision Variable:

  • assignment[i]: Doctor index assigned to patient i (0 to nD-1)

Optimization Algorithms

Random Solver

Type: Baseline / Stochastic Search

Algorithm Flow:

  1. Generate random assignment: each patient randomly assigned to a doctor (0 to nD-1)
  2. Calculate TF (standard deviation of doctor workloads)
  3. Repeat for specified iterations, keeping the best solution found

Characteristics: No intelligent search, purely random exploration. Serves as performance baseline.

Greedy Solver

Type: Deterministic / Constructive Heuristic

Algorithm Flow:

  1. Initialize all doctor loads to 0
  2. For each patient (in order):
    • Find doctor with minimum current workload
    • Assign patient to that doctor
    • Update doctor's workload
  3. Calculate final TF

Characteristics: Fast, deterministic (always same result for same input), myopic (makes locally optimal choices).

DE Solver (Differential Evolution)

Type: Population-based Evolutionary Algorithm

Algorithm Flow:

  1. Initialization: Generate 80 random individuals (assignments)
  2. Evaluation: Calculate fitness (TF) for each individual
  3. For each generation:
    • Mutation: For each target individual i:
      • Select two random individuals r1, r2 (different from i)
      • Create mutant vector: best + F Ã— (r1 - r2)
      • F = 0.8 (scaling factor)
    • Crossover:
      • Combine target and mutant using crossover rate CR = 0.9
      • For each gene: use mutant gene with probability CR, else use target gene
      • Discretize result (round to valid doctor index)
    • Selection:

      • Update global best if improved
  4. Return best solution found

Parameters: PopSize=80, F=0.8, CR=0.9, Generations=100

GA Solver (Genetic Algorithm)

Type: Population-based Evolutionary Algorithm

Algorithm Flow:

  1. Initialization: Generate 80 random individuals
  2. Evaluation: Calculate fitness for each individual
  3. For each generation:
    • Tournament Selection:
      • Randomly select 2 individuals
      • Choose the one with better fitness as parent
    • Reproduction: Create child by copying parent genes
    • Mutation:
      • With probability MR = 0.1: randomly change one gene to a random doctor
    • Evaluation: Calculate child fitness
    • Repeat to create new population of 80 individuals
  4. Track and return global best across all generations

Parameters: PopSize=80, MR=0.1, Generations=100

DGA Solver (Distributed Genetic Algorithm)

Type: Distributed Population-based Evolutionary Algorithm

Algorithm Flow:

  1. Initialization: Create 4 sub-populations (NSP=4), each with 20 individuals (SPS=20)
  2. Evaluation: Calculate fitness for all individuals in all sub-populations
  3. For each generation:
    • Local Evolution (for each sub-population independently):
      • Tournament Selection: Select 2 pairs of individuals, choose better from each pair as parents (2 parents total)
      • Crossover: Create child by randomly selecting genes from parent 1 or parent 2 (50% probability each)
      • Mutation: With probability MR=0.1, randomly change one gene
      • Evaluation: Calculate child fitness
      • Repeat to create new sub-population of 20 individuals
    • Migration (every MI=5 generations):
      • Extract best individual (elite) from each sub-population
      • Send each elite to the next sub-population (circular: 0â†’1â†’2â†’3â†’0)
      • Replace a random non-best individual in target sub-population with incoming elite
      • Purpose: Share good solutions between sub-populations, maintain diversity
    • Global Best Update: Find best individual across all sub-populations
  4. Return global best solution

Parameters: NSP=4, SPS=20, MR=0.1, MI=5, Generations=100

Advantages: Parallel evolution maintains diversity, migration mechanism facilitates knowledge sharing, typically outperforms standard GA on complex problems.

Research Objectives

Primary Objective

Compare the effectiveness of five optimization algorithms (Random, Greedy, DE, GA, DGA) on cloud healthcare patient-doctor scheduling across 16 test instances (T1-T16) with varying problem scales.

Expected Outcomes

Demonstrate that DGA (Distributed Genetic Algorithm) achieves superior performance in terms of solution quality (lower TF values) and consistency across different problem instances.

Evaluation Metrics

Performance Measures:

  • Average TF: Mean objective function value over multiple runs (lower is better)
  • Standard Deviation: Consistency of results across runs (lower indicates more stable algorithm)
  • Convergence Speed: How quickly the algorithm reaches good solutions (visualized in convergence graphs)
  • Scalability: Performance on different problem sizes (100-400 patients, 10-40 doctors)
